OVERVIEW

GOALS

Didactical Goals:
  • Evaluate Server Performance: in a simple way.
  • Sockets Usage: for learning software communication.
  • Threads Usage: almost all classes are multithread
Advantages:
  • Clear Architecture: really appropiate for didactical purposes.
  • Massive debugging: with several levels of details, and one log per connection, one can understand the whole server interactions.
Known Issues:
  • Too heavy, too slow: the architecture is based in multithread classes, for having just one object running all the time. But, in order to have better debugging, it actually open up new objects per connection, making it too slow. This can be easily fixed, since it is not though to work this way, I just use for watching better the server performance.
  • Just Text: images breaks the socket pipe, since is using String Java Vectors for buffering.
  • Simple socket management: not using a pool of sockets, meaning expensive connections.
  • Too many loggers: it keeps one Logger per connection.

INSTALLATION

Requirements:
  • Ant/Java: for executing it.
  • Elipse: for visual access to the code.
Instructions:
  • Download and Unpack the source code.
  • Build and Launch with ant:
    cd ${PROJECT_PATH}
    ant
  • http://localhost:8000 in a browser.
  • Check the logs for seeing how it works.

Browsers: IE7 won't render fine, due to IE image processing bug. Please use IE8, Opera 9+, Firefox 3+ or Chrome.
