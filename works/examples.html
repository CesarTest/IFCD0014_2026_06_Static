From my point of view, all IT systems (from complex networking IT systems that involved management, communication and so on, to small standalone programs) can be understood using software modelling techniques (as "roles designs and interactions" ), in a very clean way. How you implement each role involved in the chain and the way they interact (between all elements inside the system, and with other outside systems) is another issue, opening up a world of technologies. From signal theory for HW design, passing through the broad set of protocols and the enormous range of SW technologies.

In many cases, standarization come into scene -as a way to accumulate the experience of people in the designs, and clear interfaces between roles- and Open Source -as a set of tools for designing more complex systems-. If I need to build up a complex IT system, it is cheaper to pay for the required pieces, instead of redesigning them (and all the consequences of the design: support, mantainance, updates, etc.). This way, much more needs can be covered reassembling properly the same pieces. So, as IBM slogan, the future is open.

Since software modelling is, for me, the heart of all this IT world, this is what I would like to study systematically for now on, on a very practical way.
